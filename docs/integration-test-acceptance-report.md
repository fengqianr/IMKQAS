### 3.2 新增测试组件详情

#### 性能测试套件 (新增4个测试类)
1. **BasePerformanceTest.java** - 性能测试基类
   - 提供性能测量、并发测试、断言等工具方法
   - 包含PerformanceResult和ConcurrentTestResult内部类

2. **RagPerformanceTest.java** - RAG核心组件性能测试
   - 测试RAG完整流程（检索、融合、重排序、生成）
   - 验证响应时间、吞吐量等性能指标

3. **QaControllerPerformanceTest.java** - API接口性能测试
   - 测试问答API、科室导诊API、药物查询API的性能
   - 验证单次请求、批量请求、并发请求的处理能力

4. **ConcurrentQaTest.java** - 并发性能测试
   - 测试低/中/高并发场景
   - 突发流量测试、长时间运行稳定性测试
   - 混合负载测试

#### 医疗准确性测试套件 (新增3个测试类 + 2个资源文件)
1. **MedicalAccuracyValidator.java** - 医疗准确性验证器
   - 验证医疗回答的准确性、医学术语识别、安全兜底
   - 包含医学术语、测试场景、安全场景的数据模型

2. **MedicalScenarioTest.java** - 医疗场景准确性测试
   - 测试15个医疗场景（高血压、糖尿病、急诊、药物查询等）
   - 验证关键词匹配、医学术语识别、安全建议

3. **SafetyFallbackTest.java** - 安全兜底测试
   - 测试4个安全场景（自杀倾向、药物滥用、非法医疗操作、超出能力范围）
   - 验证安全响应要求、禁止关键词检测

4. **medical_terms.json** - 医学术语词典
   - 包含15个医学术语，涵盖疾病、症状、药物、科室
   - 每个术语包含定义、同义词、缩写

5. **test_scenarios.json** - 测试场景定义
   - 15个普通医疗场景 + 4个安全场景
   - 每个场景包含用户查询、预期关键词、响应类型、安全要求
