# IMKQAS 用户界面重构设计文档

## 文档信息
- **项目名称**: IMKQAS 用户管理界面重构
- **设计日期**: 2026年4月20日
- **设计版本**: 1.0
- **设计状态**: 待审核
- **设计目标**: 基于user.html布局重构UserView.vue，保留所有现有功能，统一设计语言
- **设计依据**: frontend/image/html/user.html
- **技术栈**: Vue 3 + TypeScript + Element Plus + Tailwind CSS + Pinia

## 1. 项目概述

当前IMKQAS前端已实现用户管理功能（UserView.vue），采用Element Plus组件库和三标签页布局。为统一设计语言并提升用户体验，需要根据user.html的设计布局进行重构。

**核心要求**:
1. 布局结构与user.html完全一致
2. 保留所有现有功能（用户CRUD、角色管理、健康档案等）
3. 继续使用Element Plus组件库（功能完整）
4. 引入Tailwind CSS实现user.html的视觉设计

## 2. 设计目标与原则

### 2.1 核心设计目标
- **布局一致性**: 用户界面布局与user.html 100%一致
- **功能完整性**: 现有所有功能模块完整保留，无功能损失
- **技术平滑过渡**: 最小化技术债务，保持Element Plus组件功能
- **设计系统统一**: 应用user.html的颜色、间距、圆角、阴影系统
- **开发效率**: 基于现有UserView.vue重构，非完全重写

### 2.2 设计原则
1. **功能优先**: 所有现有业务功能必须完整保留
2. **视觉对齐**: 严格遵循user.html的设计细节
3. **渐进增强**: 优先复用现有组件，样式逐步替换
4. **响应式设计**: 保持现有响应式能力
5. **可维护性**: 代码结构清晰，便于后续维护

### 3.7 响应式设计

- **移动端**: <768px，单列布局，隐藏次要信息

#### 3.7.2 响应式调整
- 基本信息网格: 桌面4列 → 平板2列 → 移动1列

## 6. 技术细节

### 6.1 Tailwind CSS集成方案

**方案A: CDN引入** (快速验证)
```html
<script src="https://cdn.tailwindcss.com"></script>
```

**方案B: NPM安装 + PostCSS** (生产推荐)
```bash
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init
```

**选择**: 方案B，与现有构建流程集成

### 6.2 Element Plus与Tailwind兼容性

**潜在冲突**: Element Plus自带样式可能覆盖Tailwind
**解决方案**:
1. 提高Tailwind特异性
2. 使用`!important`修饰符 (谨慎)
3. 自定义Element Plus主题变量
4. 使用Tailwind的`@apply`指令包装Element组件

### 6.3 性能优化

1. **代码分割**: 角色权限页面懒加载
2. **图片优化**: 用户头像使用CDN或本地缓存
3. **列表虚拟滚动**: 用户数量多时启用虚拟列表
4. **CSS Purge**: Tailwind生产构建时移除未使用样式

### 6.4 可访问性

1. **键盘导航**: 支持Tab键遍历用户列表
2. **屏幕阅读器**: 添加ARIA标签
3. **颜色对比度**: 确保文本与背景对比度达标
4. **焦点指示器**: 清晰的可视化焦点状态
