## 项目概述

### 项目目标
构建基于RAG（Retrieval-Augmented Generation）架构的医疗知识智能问答系统，为医学生、基层医护、普通患者和健康管理师提供专业、准确、安全的医疗知识查询服务。

### 核心价值
- **准确性**: 基于权威医学文献，避免大模型幻觉
- **安全性**: 医疗安全兜底机制，紧急症状提醒就医
- **专业性**: 医疗术语规范化，科室导诊分流
- **可视化**: 检索过程可视化展示技术深度
- **易用性**: 对话式交互，流式输出体验

## 系统架构

### 六层架构
1. **展示层**: Vue3 + Element Plus + ECharts (响应式前端)
2. **网关层**: Spring Cloud Gateway (统一入口、限流、鉴权)
3. **应用层**: SpringBoot + Langchain4j (核心业务逻辑)
4. **RAG引擎层**: 多路召回 + RRF融合 + Cross-Encoder重排序 (系统亮点)
5. **模型服务层**: BGE-large-zh + BGE-reranker-v2-m3 + 医疗LLM
6. **存储层**: MySQL + Milvus + Redis + MinIO

## 核心功能模块

### 1. 智能问答模块 (核心)
- **流式对话**: SSE推送，类似ChatGPT打字机效果
- **来源溯源**: 每条回答标注引用的医学文献
- **多轮上下文**: Redis存储上下文，LLM指代消解
- **医疗安全兜底**: 置信度低时提示"建议就医"
- **科室导诊分流**: 症状描述→科室建议 (答辩加分项)
- **药物查询**: 药品说明书检索，药物相互作用检查
- **检索可视化**: 展示完整RAG链路 (答辩演示专用)

### 2. 知识库管理模块
- 文档上传 (PDF/Word/TXT)
- 智能切分 (文本提取→结构化切分→向量化→入库)
- 分类管理 (内科/外科/儿科/妇科等)
- 文档预览与切片浏览

### 3. 用户管理模块
- 手机号+验证码登录 (JWT鉴权)
- 两种角色: 医护人员(知识库+问答)、普通用户(问答)
- 健康档案: 年龄/性别/过敏史/慢病史，问答时注入Prompt

### 4. 对话管理模块
- 多会话管理 (类似ChatGPT侧边栏)
- 历史记录搜索
- 会话导出 (PDF/Markdown)
- 回答收藏与反馈

### 5. 数据统计与可视化模块
- 问答统计图表 (ECharts)
- 热门问题排行榜
- 科室分布饼图
- 知识库覆盖统计
- 用户活跃度分析

## 技术实现细节

### RAG流程
```
用户提问 → Query改写LLM → 预处理 → 多路并行检索 → RRF融合 → Cross-Encoder重排序 → Prompt构建 → LLM生成 → 流式输出
```

### 多路检索策略
1. **向量检索**: Milvus + BGE-large-zh embedding
2. **关键词检索**: BM25算法
3. **混合检索**: 综合语义与关键词匹配
4. **RRF融合**: Reciprocal Rank Fusion算法
5. **重排序**: BGE-reranker-v2-m3医疗语义匹配

### 数据库设计
- **MySQL**: 用户、文档、对话、消息等结构化数据
- **Milvus**: 文档向量存储 (1024维，BGE-large-zh)
- **Redis**: 查询缓存、会话上下文、限流计数
- **MinIO**: 医学文档文件存储

### 限流策略
| 角色 | 默认限流 | 突发容量 |
|------|---------|----------|
| 普通用户 | 10请求/秒 | 20 |
| 管理员 | 50请求/秒 | 100 |

特殊接口: 文档上传接口限制2请求/秒 (所有角色)

## API接口

### 主要端点
- `POST /api/qa/stream` - 流式问答 (SSE)
- `POST /api/qa/triage` - 科室导诊分流
- `GET /api/qa/drug` - 药物查询
- `POST /api/kb/documents` - 上传文档 (ADMIN)
- `POST /api/auth/login` - 手机号登录
- `GET /api/stats/dashboard` - 数据大屏 (ADMIN)

## 开发计划

### 阶段划分 (共9周)
1. **阶段1**: 基础框架 (2周) - SpringBoot项目、数据库、基础API

## 风险评估与缓解

| 风险 | 缓解措施 |
|------|----------|
| 医疗准确性风险 | 严格来源标注、安全兜底、人工审核 |
| LLM API成本 | 查询缓存、答案缓存、频率限制 |
| 向量检索性能 | Milvus索引优化、结果缓存、降级策略 |
| 数据隐私安全 | 数据加密、访问控制、匿名化处理 |
| 并发压力 | 限流策略、服务降级、弹性扩容 |
